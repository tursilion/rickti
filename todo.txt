The core logo is about 39x21 and the firebird logo is also 39x21. So, again, lots of wasted space, but we'd need 3 16x16 sprites across and 2 down each, total of 12 sprites. We would need to space the sprite tables by 48 characters (12*4), that fits in 512 bytes, so the three sprite pattern tables would take 1536 bytes, leaving 2432 bytes after the SAL (128 bytes). For simplicity - we can use the same GPU program as the Rick caching uses. We'll just reserve one code for "sprite toggle", and it'll check that every frame, and do one frame of sprite flips. The CPU can then clear it when it's done. (Conversely to the Rick sprite update, which it will clear itself when it's done.) That way we don't have to race the GPU program... though I guess stopping it is easy. But none of the work to be done will take much time.
